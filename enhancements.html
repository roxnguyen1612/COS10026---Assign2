How we go beyond the requirements

Web design improvements

Responsive design is about making the website responds to different screen sizes or viewports.

Implementation:

  • Meta tags: <meta name="viewport" content="width=device-width, initial-scale=1.0":>:
  • This will set the viewport for the web page, which will give the browser instructions on how to control the page’s dimensions and scaling

Media queries:

  • defines different styles for different browser sizes. In our index.html, we used media queries for different sizes:
  • @media (max-width: 572px): When the device’s screen width fall below 572px, it will apply the CSS inside the media query
  • @media (min-width: 800px): When the device’s screen width go beyond 800px, it will apply the CSS inside the media query.

Text reveal animation:

  • keyframes rule(@keyframes):
  • specifies the animation code.
  • specify when the style change will happen in percent
  • animation: bring an animation to a div element

In order to improve the web design when the screen width is smaller, we have implemented:

  • Changes to the text font sizes so that it’s more readable.
  • Change the background when the screen width fall below 572px
  • Replace the page links with a hamburger menu button.
  • Since the assignment doesn’t allow JavaScript, we used the checkbox instead, so that when it’s checked, the screen will display a menu that contains all the links to other html pages.

Several CSS properties that goes beyond the basic requirements that we used for the hamburger menu:

  • z-index: specifies the stack order of an element. If the stack order of an element is higher than the others, it will be in front of others.
  • transform: applies a 2D or 3D transformation to an element.
  • border-radius: allow us to add rounded corners to elements.
  • CSS selector “~” (element1 ~ element2): Selects every “element1” that is preceded by an “element2” element.

Bootstrap5 implementation

To improve the web design, we have used the Bootstrap framework. We used Bootstrap 5 since it is compatible with all modern browsers (Chrome, Firefox, Safari, etc…) and relatively easy to use.

We have included Bootstrap 5 from a CDN:

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/css/bootstrap.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/js/bootstrap.bundle.min.js">

This is the list of our bootstrap implementation:



Checks/radio bootstrap buttons:

  • Uses a wrapper element with class=”form-check” to ensure proper margins for labels and checkboxes.
  • Add the .form-check-label class to label elements
  • Add the .form-check-input to style checkboxes.

Submit/reset bootstrap buttons

  • By using bootstraps custom button styles, it allows for consistent styling with the rest of the form.
  • The “btn-success” class is used for the submit button.
  • The reset button is less important and so the “btn-secondary” class is used.

Floating labels for the text input fields:

Making the labels float/animate when you click on the input field:

  • We have implemented it for input fields, textareas, select menus.

Form-control class for proper styling:

  • The “form-control” class give input/textarea elements custom styling, sizing and focus states.
  • Allows for consistent styling across the webpage.

Bootstrap style validation:

  • The “was-validated” class is applied to the form element. This allows the use of bootstraps custom style validation messages on input elements within the form.
  • For custom validation messages, div elements with the “invalid-feedback” class applied to them, are placed under input, select and textarea elements.
